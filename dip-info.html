ABOUT DIP SWITCH DATA

Where the data comes from

DIP switch data on ArcadeRTFM comes from two sources. The majority is sourced from the MAME (Multiple Arcade Machine Emulator) project's machine database. Some entries also contain DIP switch information extracted from the original service manuals using LLM vision analysis of the scanned pages. Where both sources are available, the data is combined.

The early era: DIP switches controlled everything

On hardware from the late 1970s through the late 1980s, DIP switches were the primary way to configure a game. Difficulty, number of lives, coinage, bonus thresholds, cabinet type, language. It was all set via physical switches on the PCB. For these machines, the DIP tables shown on ArcadeRTFM are the complete configuration reference. This covers most of the classic era: Pac-Man, Galaga, Donkey Kong, Defender, Robotron, and hundreds of others.

The transition: later hardware moved to service menus

From the early 1990s onward, most arcade hardware shifted game configuration to software-based Test/Service Menus, with settings saved to battery-backed NVRAM. On these machines, you access settings by pressing the Test or Service button inside the cabinet, navigating an on-screen menu, and the board remembers the changes after power-off.

DIP switches still exist on some of these later boards, but they typically control low-level hardware functions rather than game settings: debug modes, graphics layer enables, and test features that were used during development.

The Model 2 / Versus City quirk

Sega's Model 2 platform (and some Model 3 and ST-V games) has a specific known quirk that affects how DIP data appears in MAME and therefore on this site.

These systems supported an optional accessory called the Versus City Billboard, a separate display unit mounted on top of linked cabinets for head-to-head play. The billboard unit had its own 6-8 position DIP switch pack controlling test LEDs, 7-segment display tests, and demo mode.

MAME exposes the billboard DIP switches in its standard DIP menu, which means they appear in our database. However, the main game board's own DIP switches (typically labeled SW3 on the PCB) are either hidden, labeled as "Unknown", or filed under a different internal port name in MAME. These main board switches mostly control debug features and have little effect on normal gameplay.

In practice, this means:

DIP TypeControlsShown on ArcadeRTFM?
Billboard DIPsTest LEDs, demo mode, 7-segment test patternsYes (from MAME)
Main board DIPsDebug features, hidden test modesNo (hidden/unnamed in MAME)
Game settingsDifficulty, coinage, timer, free play, etc.No (configured via Test Menu, stored in NVRAM)

This is not an error in our data. It accurately reflects what MAME documents. The game settings that operators actually need to change (difficulty, coinage, round timer) were never configured via DIP switches on Model 2 hardware. They are set through the on-screen Test Menu accessed via the service button.

Why some switches are shown as "Unknown"

The DIP switch table shows every switch position that MAME documents, including those labeled "Unknown" or "Unused". These are shown dimmed in the table. We include them rather than hiding them because the physical DIP switch pack on the board has a fixed number of positions, and if we skipped the unknown ones, the remaining entries would appear to be in the wrong positions. Someone reading "Coinage" as the third row might assume it is physical switch 3, when it is actually switch 5 because switches 3 and 4 are undocumented.

Each row shows the physical switch location (e.g. SW1:3, SW2:7) in the first column so you can match the setting to the correct physical switch on the board. The "Unknown" entries typically have no observable effect during normal operation. In some cases they were factory test points, in others they were reserved for regional variants that were never produced, and in many cases nobody has yet reverse-engineered what they do.

Which machines are affected?

This primarily affects Sega Model 2, Model 3, ST-V, Naomi, and similar later platforms from the mid-1990s onward. It also applies to many Namco System boards, Konami boards, Capcom CPS-3, and most other hardware from the 3D polygon era. If a machine is from roughly 1993 or later and shows only a handful of DIP switches with options like "Off/On" and test-related names, the actual game configuration is almost certainly in a service menu.

General guidance

For pre-1990 hardware: DIP tables on this site are likely the complete game configuration reference. Cross-reference with the service manual if one is available.

For post-1993 hardware: DIP switches shown are probably hardware-level or accessory settings. Look for the Test/Service button on the cabinet to access the actual game settings menu.

For the early 1990s transition period: it varies by manufacturer and platform. Some boards used both DIP switches and service menus. Check the manual if available.
